## 2026-06-11 · Интерактивный лендинг (landing.css v9, landing.js v1)

### Стек анимаций: vanilla JS + CSS, без библиотек
**Решение.** Вся интерактивность — на IntersectionObserver, requestAnimationFrame
и CSS-анимациях (`transform`/`opacity` только).
**Почему.** Лендинг — Jinja2-шаблон без сборщика; любая библиотека (Framer,
GSAP, Lenis) — это +30–90 КБ JS, внешняя зависимость и удар по Lighthouse.
Весь нужный набор эффектов покрывается ~9 КБ собственного JS.
**Отклонено.** Lenis для smooth scroll — нативный `scroll-behavior: smooth`
уже включён и достаточен; Lenis перехватывает скролл и часто вредит
доступности.

### Hero: живое чат-демо вместо статичного макета
**Решение.** Телефон в hero проигрывает сценарий: typing-индикатор → напоминание
→ пульсирующие кнопки «Буду/Отменяю» → ответ ученика → ответ бота → карточка
«Репетитору: Маша подтвердила…» → кнопка «Показать ещё раз». Цикл ~9 с.
Кнопки настоящие: посетитель может кликнуть сам (в т.ч. «Отменяю» — покажет
ветку отмены); если не кликает — демо отвечает за него на 6-й секунде.
**Почему.** Продукт — это и есть этот диалог. Показ цикла «бот → ученик →
репетитор» объясняет ценность быстрее любого текста, а карточка
«Репетитору: …» демонстрирует вторую половину продукта (статус у репетитора),
которой в старом макете не было вовсе.
**Детали против layout shift.** Лента чата имеет фиксированную высоту,
уведомление и кнопка повтора зарезервированы по месту (скрыты через
opacity, не display) — телефон не «прыгает» во время анимации, CLS = 0.
Лента прижата к низу (justify-content: flex-end): диалог растёт вверх, как
в настоящем мессенджере, а при переполнении обрезаются старые сообщения,
а не последние. Автозапуск карусели отзывов — тоже только в viewport.
**Reduced motion.** Демо не проигрывается — сразу показывается финальное
состояние диалога.

### Запуск демо по видимости
Демо стартует, когда телефон попадает в viewport (threshold 0.4), а не по
load — на мобильных телефон ниже первого экрана, и анимация не должна
отыграть «вхолостую».

### «Как это работает»: reveal + счётчики + линия + tilt
**Решение.** Карточки появляются каскадом (stagger 0.12 c через CSS-переменную
`--d`), номера тикают 00→01/02/03, между карточками «прорисовывается»
пунктирная линия (scaleX по `:has(.step.in)` — без лишнего JS), на hover —
3D-tilt максимум 5°.
**Почему.** Последовательное появление + линия подчёркивают, что это именно
последовательность шагов, а не три равноправные фичи.
**Отклонено.** SVG-стрелки между карточками — при grid-раскладке их позиционирование
хрупкое; горизонтальная линия на уровне иконок даёт тот же эффект дешевле.
**Деталь.** Reveal сделан анимацией (keyframes), а не transition — иначе
transition по transform конфликтовал бы с inline-transform от tilt
(каждое движение мыши анимировалось бы 0.6 с).

### «Почему Pingly»: hover-подъём + анимированные иконки
**Решение.** Hover: translateY(-4px), оранжевый glow + подсветка границы +
радиальный градиент в углу (через ::before, только opacity). Иконки имеют
персональные микроанимации (молния вспыхивает, курсор «тапает», сообщение
покачивается, щит пульсирует): один раз при появлении карточки и повторно
при hover.
**Деталь.** Reveal-анимация вешается на обёртку `.feat-ic`, hover-анимация —
на `svg` внутри: иначе одинаковое имя анимации не перезапускается.

### «Возможности»: табы с мини-макетами вместо сетки 6 карточек
**Решение.** 6 фич — в табы. Подчёркивание «переползает» (JS двигает один
индикатор по offsetLeft/width). Старая панель уезжает влево с fade, новая
въезжает справа. Панели лежат в одной grid-ячейке — высота блока равна
максимальной панели, переключение не дёргает страницу. В каждой панели —
стилизованный мини-UI из div'ов (список оплат, мини-визитка с заявкой,
чат переноса, неделя календаря, чек-лист ДЗ, карточка ученика).
**Почему.** Сетка из 6 одинаковых текстовых карточек читается как «стена фич».
Таб с мини-макетом показывает, как фича выглядит, а не рассказывает.
Мини-UI из div'ов вместо скриншотов: ничего не весит, не устаревает при
изменении продукта, остаётся в дизайн-системе.
**A11y.** role=tablist/tab/tabpanel, aria-selected, стрелки ←/→, roving
tabindex. На мобильных — горизонтальный скролл с scroll-snap.

### «Отзывы»: карусель вместо сетки
**Решение.** Один отзыв на экран (max-width 640), slide+fade трека,
автопрокрутка 5 с с паузой на hover/focus и при скрытой вкладке, кликабельные
точки (активная вытягивается в полоску). Аватарки плавно «плавают».
**Почему.** Отзывов пока три; в сетке они выглядят скромно, карусель
фокусирует на одном отзыве и добавляет жизни секции.

### Глобальное
- **Прогресс-скролл**: 3px градиентная линия сверху (scaleX по rAF).
- **Magnetic CTA**: главные оранжевые кнопки притягиваются к курсору
  (зона = кнопка + 50px, смещение ≤ ~12% расстояния). Один listener на
  document + rAF; отключено на touch и при reduced motion.
- **Shimmer**: блик по primary-кнопкам на hover (::after, translateX).
- **Фон hero**: два размытых оранжевых блоба с медленным дрейфом
  (transform-only). На мобильных скрыты — там они только жгут GPU.
- **Хедер**: glassmorphism усилен (blur 14px + saturate 160%), при скролле
  появляется тень.
- **H1** укрупнён (clamp до 3.6rem), letter-spacing −0.025em.

### Производительность и доступность (общие ограничения)
- Анимируются только `transform` и `opacity`; `will-change` — точечно
  (прогресс-бар, индикатор табов, трек карусели, блобы, tilt на время hover).
- `prefers-reduced-motion`: глобальный kill-switch уже в design-tokens.css;
  поверх него JS не запускает демо/tilt/magnetic/автокарусель, а CSS
  гарантирует, что reveal-контент не останется скрытым.
- Без JS лендинг полностью читаем: скрывающие стили живут под `html.js`,
  плюс страховочный таймер (4 с) дораскрывает всё, если IO не сработал.
- Кэш: `landing.css?v=9`, `landing.js?v=1` — CDN отдаёт /static с долгим
  Cache-Control, версия в query обязательна при каждом изменении.

### Технические ограничения, с которыми столкнулись
- `/static` кэшируется CDN надолго — любое изменение CSS/JS требует bump
  версии в query-строке шаблона.
- `landing.css` делится с auth-страницами (/login, /register) и публичной
  визиткой — секции Auth/Public менять при редизайне лендинга нельзя.
- `prefers-reduced-motion` в design-tokens.css обнуляет длительности всех
  анимаций глобально — любые «скрыто до анимации» состояния обязаны иметь
  reduced-motion-фолбэк, иначе контент исчезнет.

## 2026-06-11 · UX-редизайн дашборда (app.css v42, app.js v1)

### Инфраструктура: window.pingly вместо копипасты по шаблонам
**Решение.** Новый `web/static/app.js` — единый рантайм кабинета: тосты
(`pingly.toast`, 4 семантических типа, автозакрытие 4 с, ошибки — липкие),
буфер обмена с фидбеком, тикающие счётчики `[data-count]`, относительное
время `[data-reltime]` («через 2 часа», обновляется раз в минуту),
stagger-появление карточек, спиннер на submit-кнопках, Escape/backdrop/
focus-trap для всех модалок, collapse сайдбара, микро-конфетти.
**Почему.** До этого каждый шаблон копировал свой copyField/escape-обработчик.
Единый рантайм = одинаковое поведение везде и меньше кода в шаблонах.
**Деталь.** app.js подключён в `<head>` синхронно (не defer): инлайн-скрипты
страниц зовут `pingly.*` при парсинге — с defer они падали бы (поймано
Playwright-тестом).

### Семантические цвета — закреплены как конвенция
Зелёный = успех/оплачено, красный = отмена/долг, синий = запланировано/инфо,
оранжевый = действие/CTA, серый = неактивно. Бейджи статусов уже следовали
этой схеме; тосты, бейдж «просрочено», блоки «не оплачено» приведены к ней же.

### Сайдбар: collapse + контрастный активный пункт
Активный пункт — сплошной оранжевый фон + белый текст (было — бледный
orange-50, терялся). Кнопка «Свернуть» прячет подписи (остаются иконки с
title-тултипами), состояние в localStorage и читается до первой отрисовки —
без мигания. На планшетах/мобильных кнопка скрыта (там свой нав).

### Обзор: каждый блок ведёт к действию
- Метрики — теперь ссылки в свои разделы (hover-lift + стрелка), числа
  тикают 0→N. Мини-тренды «за месяц» отклонены: исторических данных в БД
  нет, рисовать фейковые стрелки хуже, чем не рисовать.
- «Ближайшие занятия»: однотипные бейджи «Запланировано» заменены на
  относительное время («через 2 часа» — оранжевый, если скоро); занятия
  сегодня подсвечены фоном; на hover — «Перенести →» в день календаря.
- «На проверку» пустой — инлайн-форма «Выдать задание» (ученик + название +
  описание) прямо в карточке, POST в существующий /tutor/homework.
- Финансы: «Заработано» крупно, «Не оплачено» — красный блок с
  «Собрать оплату →»; пустые заявки — CTA «Настроить страницу заявок»
  с мягким пульсом (3 цикла, не бесконечно).

### Ученики: статус → действие
- «не подключён» заменён кнопкой «Отправить ссылку» (копирует инвайт + тост
  «отправь ученику в Telegram»). Бейдж остался только у подключённых.
- Pill-фильтры: все / в Telegram / не подключены + по предметам (если их >1).
  Клиентская фильтрация — список уже на странице, ходить на сервер незачем.
- Поиск live с debounce 300 мс и подсветкой <mark>; «ничего не найдено» →
  CTA «Добавить ученика „имя"» с префиллом модалки.
- Удаление спрятано в «⋯»-меню (редкое и опасное), правка и ссылка — видимые.
- Пустой список — онбординг из 3 шагов с прогресс-баром.

### Календарь: планировщик, а не просмотрщик
- «Сегодня» — оранжевый фон + бейдж (раньше только рамка).
- В месяце ячейка показывает 2 занятия + «+N ещё» → поповер со списком и
  ссылкой «Открыть день» (не уводим со страницы ради просмотра).
- Hover пустой ячейки — «+»: мини-модалка (ученик + время) бьёт в
  существующий POST /tutor/schedule (recurrence=once); добавлен form-параметр
  `back`, чтобы вернуться в тот же вид календаря, а не в month.
- Легенда — кликабельные фильтры статусов (toggle show/hide).
- Переключатель День/Неделя/Месяц — segmented control с подъезжающим
  тамбом. Полный shared-layout «переполз» между страницами невозможен без
  SPA — тамб анимируется при загрузке, направление листания месяца
  передаётся через sessionStorage (слайд влево/вправо).
- Переходы prev/next — лёгкий slide контента по направлению.

### Финансы: управляемые, а не отчётные
- Чекбоксы у неоплаченных + плавающий action bar («Выбрано N · сумма» →
  «Отметить оплачено»). Массовая отметка — последовательные fetch в
  существующий /tutor/lessons/{id}/paid, строки уезжают вправо, тост с
  суммой, микро-конфетти 0.6 с, reload.
- Sparkline дохода за 6 месяцев — данные считаются в роуте из уже
  загруженных занятий (0 лишних запросов к БД); период «месяц/квартал/всё»
  переключается на клиенте из того же JSON.
- Неоплаченные отсортированы старые-сверху + бейдж «просрочено N дн.»
  (>7 дней, считается на клиенте от data-ts).
- «Напомнить» копирует готовое сообщение об оплате. Прямая отправка в TG
  ученику отклонена: это новый бэкенд-функционал (вне границ задачи) и
  спорный UX — репетитор должен видеть, что отправляет.
- Клик по ученику — раскрывающаяся история последних занятий (оплачено/долг).

### Заявки: предпросмотр первичен
Split-view: слева настройки, справа sticky-превью публичной страницы
(аватар, имя, предметы, плашки, «О себе», кнопка) — обновляется на каждый
ввод. Тема — три thumbnail-радио вместо select. «О себе» — счётчик 600.
@username в заявках автоматически становится ссылкой t.me. QR-код отклонён:
без библиотеки не сделать честно, внешний QR-сервис = утечка ссылок.

### Настройки
Рефералка вынесена из подвала подписки в отдельную карточку с кнопками
«поделиться» в Telegram/WhatsApp (share-URL, без SDK). Прогресс «приведено N»
отклонён — счётчика рефералов нет в контексте роута; добавить = трогать
бэкенд. Тарифы: бейдж «Ваш тариф», недоступные фичи Pro — title-тултип
«Доступно на тарифе Max». «Проверить уведомления» (тест-сообщение в бот)
отклонено — требует нового эндпоинта.

### Глобальное поведение
- Модалки на мобильных — bottom sheet (выезжают снизу, скруглён только верх).
- Скелетоны отклонены: всё SSR, асинхронных списков нет — скелетону нечего
  ждать. Вместо этого спиннеры в кнопках на submit.
- Reduced motion: счётчики показывают финал сразу, конфетти/пульс/слайды
  выключены, поверх глобального kill-switch из design-tokens.css.

### UX-гипотезы для проверки метриками
- Массовая отметка оплат сократит время работы с финансами (~10 кликов → 2).
- «Отправить ссылку» поднимет долю подключённых учеников (сейчас 2 из 11).
- Live-превью страницы заявок увеличит долю включивших публичную страницу.
- Относительное время в «Ближайших занятиях» снизит пропуски «горящих» слотов.

### Ограничения, встреченные в этой итерации
- Jinja2 без list comprehensions и фильтра zip — данные для поповера
  календаря собираются через {% set %} + append + tojson.
- tour.js (онбординг-тур) перехватывает клики оверлеем на «свежем»
  localStorage — в e2e-тестах нужно сетить pingly_tour_v1=1 до загрузки.
- fin_history содержит и оплаченные, и неоплаченные занятия — «за всё время»
  на клиенте = сумма всех проведённых (консистентно с month_earned сервера).
