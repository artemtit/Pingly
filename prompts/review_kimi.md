# Kimi — полная проверка кода и безопасности Pingly

Kimi силён в длинном контексте и рассуждении. Дай ему **весь** релевантный код разом (см. список файлов внизу), затем вставь промпт. Если код не помещается — грузи по слоям: сначала `web/app.py` + `infrastructure/supabase_repository.py` + `config.py`, затем `application/services/*`, затем `infrastructure/platega.py` + `bot.py` + миграции.

---

Ты — независимый аудитор безопасности. Перед тобой исходники продакшен-SaaS **Pingly** (напоминания + CRM для репетиторов): хранит ПДн репетиторов/учеников, принимает платежи через Platega (СБП), пускает пользователей через Telegram. Проведи **глубокий аудит безопасности и корректности**, рассуждая по фактическому коду, который тебе дали. Не предполагай содержимое файлов, которых нет в контексте, — если чего-то не хватает для вывода, прямо скажи, какой файл нужен.

## Стек и модель доступа
- Python; **aiogram 3** (бот) + **FastAPI/Starlette** (веб) + **Jinja2**; **Supabase/PostgreSQL** (доступ только через `infrastructure/supabase_repository.py`); **APScheduler**; за Cloudflare.
- Слои: `web/app.py` → `application/services/*` → `infrastructure/supabase_repository.py`.
- Ключевой инвариант: **репетитор имеет доступ только к своим ученикам/занятиям/ДЗ/заявкам**. Нарушение этого инварианта = критичная уязвимость.

## Проверь и обоснуй кодом каждую зону
1. **Изоляция арендаторов / IDOR.** Для КАЖДОГО роута `/tutor/...`, принимающего `id` из URL, и для КАЖДОГО метода репозитория: фильтруется ли выборка/мутация по `tutor_user_id` текущего пользователя? Построй таблицу «роут → метод репозитория → есть ли scoping по владельцу». Отметь любой путь, где подстановка чужого UUID даёт доступ. Учти: с сервисным ключом Supabase RLS обходится — нельзя полагаться только на политики из `migrations/006_growth_rls_policies.sql`.
2. **Вебхук Platega** (`infrastructure/platega.py` + колбэк-роут): верификация подписи/секрета ДО выдачи подписки; идемпотентность; защита от replay; сверка суммы и статуса; нельзя ли активировать подписку поддельным запросом.
3. **Сессии/куки** (`itsdangerous`): HttpOnly/Secure/SameSite/TTL, что именно подписано, ротация при входе, фиксация сессии. (Контекст: фикс ISS-001 про куки на локальном HTTP — не ослаблен ли Secure в проде.)
4. **Telegram-вход:** проверка HMAC Login Widget бот-токеном и `auth_date`; одноразовость и энтропия инвайт-токенов `inv_<...>`; в боте — авторство нажатий `lesson_confirm`/`lesson_cancel` (тот ли ученик), TTL и изоляция `_awaiting_reason`.
5. **Админка:** все `/admin/...` под проверкой `is_admin`; нет ли привилегированных действий, утёкших обычному пользователю.
6. **Инъекции/XSS:** raw-SQL c f-строками; `| safe` в Jinja (бейджи/`badge_svgs`) и попадание туда пользовательского ввода → stored XSS на `/u/<slug>`; открытые редиректы (`back`/`next`).
7. **Неавторизованные формы** (регистрация, `/u/<slug>/book`): rate-limit, антибот (`CAPTCHA_ENABLED`), валидация длин, защита от спама.
8. **Секреты:** утечки в логи/репозиторий/шаблоны; корректный забор из env (`config.py`).
9. **Флаги фич:** `PAYMENTS_ENABLED`/`PLANS_ENABLED`/`PAYWALL_ENABLED`/`EMAIL_VERIFICATION_ENABLED`/`CAPTCHA_ENABLED` — обход пейволла через `_PAYWALL_OPEN_PREFIXES`/`subscription_info`; серверные проверки, а не только UI.
10. **Корректность/устойчивость:** исключения, валящие процесс (бот и веб в одном процессе); дубли напоминаний в APScheduler; путаница таймзон (МСК/UTC); тяжёлые `select("*")`/N+1.

## Формат ответа
1. **Сводка рисков** — 3–5 строк: насколько всё плохо/хорошо в целом.
2. **Таблица находок**, по убыванию серьёзности:

| ID | Severity | Категория | Файл:строка | Описание | Сценарий эксплуатации | Рекомендация |

3. Для Critical/High — фрагмент уязвимого кода + **предлагаемый патч** и почему он закрывает проблему.
4. **Матрица изоляции арендаторов** (роут → метод репо → scoping: да/нет/неясно).
5. **Топ-5 действий** в порядке приоритета.
6. **Зоны неопределённости**: где для вывода не хватило файлов/контекста (назови их), чтобы не плодить ложные срабатывания.

Будь беспощаден, но точен: только реально эксплуатируемые проблемы, с конкретными `файл:строка`. Никаких выдуманных файлов и общих фраз без привязки к коду.
